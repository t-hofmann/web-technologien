HTML

Bei HTML handelt es sich um eine Auszeichnungssprache. Teile eines Textes werden mit Hifle sogenannter Tags ausgezeichnet. Zwischen Starttag <TAGNAME> und Endtag </TAGNAME> steht der auszuzeichnende Text. Das Starttag kann zusätzlich Attribute <TAG ATTRIBUT="WERT"> enthalten, - den meisten Attributen muss ein Wert zugeordnet werden. Das class-Attribut verweist auf eine oder mehrere dem aktuellen Tag zugeordnete CSS-Klassen.

HTML-Kommentare <!-- Kommentar --> werden vom Browser nicht für die Darstellung herangezogen.

Die verschiedenartigen Elemente eines Hypertextes, wie bspw. Überschriften, Absätze, Listen, Tabellen, usw., können mit Hilfe dieser Tags ausgezeichnet werden. Im Folgenden wird ein einführender Überblick gegeben.

Ein html5-konformes Grundgerüst, - die erste Zeile <!DOCTYPE html> weist den Browser dabei auf html5 hin:

<!DOCTYPE html>
<html>
	<head>
		<!-- Metainformationen -->
	</head>
	<body>
		<!-- Darzustellender Inhalt -->
	</body>
</html>

Im head werden Metainformationen zum aktuellen Dokument untergebracht, während im body der darzustellende Inhalt steht.

Metainformationen im <head>

Der "head" einer html-Datei kann Metainformationen zum Dokument beinhalten. Diese Metainformationen können den Titel, Autor, Beschreibung und Stichworte beinhalten, genauso wie die Angabe zur verwendeten Kodierung, einzubindenden Stylesheets oder Scriptdateien. Diese Aufzählung erhebt keinen Anspruch auf Vollständigkeit. Es folgt eine Beispiel für Metainformationen im Head einer HTML-Datei: 

<head>
	<meta charset="UTF-8">

	<title>TITEL</title>
	<meta name="author" content="AUTOR">
	<meta name="date" content="ZEITANGABE NACH ISO 8601">
	<meta name="description" content="BESCHREIBUNG">
	<meta name="keywords" content="LISTE, VON, STICHWORTEN">

	<meta name="robots" content="noindex,nofollow">

	<link rel="stylesheet type="text/css" href="STYLESHEET.css">

	<meta http-equiv="refresh" content="SECONDS; URL=http://TARGETURL">
	<meta http-equiv="expires" content="0">
</head>

Ligaturen

Eine Spezialität aus dem Schriftsatz sind sogenannte Ligaturen (mittellat. ligatura "Verbindung"). Dabei handelt es sich um spezielle Schnitte um benachbarte Buchstaben zu verbinden. Durch den Einsatz von Ligaturen soll ein harmonisches Schriftbild und eine gute Lesbarkeit erreicht werden. An dieser Stelle kann nicht vertiefend auf dieses Thema eingegangen werden, deshalb sei auf den Wikipedia-Artikel: "Ligatur (Typografie)" verwiesen.

Durch den Einsatz von &zwj; ("zero width joiner") bzw. &zwnj; ("zero width non joiner") kann eine Ligatur erlaubt bzw. verhindert werden. Das folgende Beispiel setzt voraus, dass die Schriftart "Palatino Linotype" installiert ist. Für eigene Experimente könnte die Suche innerhalb der Gruppe der sogenannten "gebrochenen" Schriften hilfreich sein.

Verweise

Wesentlich für HTML sind Verbindungen (engl. link). Ein Link-Ziel kann eine beliebige i. d. R. über http erreichbare Resource sein, - https im Falle der verschlüsselten Variante. Das Hypertext Transfer Protocol -- HTTP/1.1 (Roy Fielding, et al.; RFC 2616; IETF; 1999) ist im RFC 2616 definiert. Vertiefende Einblicke liefert in diesem Kontext die Dissertation des Hauptautors "Architectural Styles and the Design of Network-based Software Architectures" (Diss.; Roy Thomas Fielding; 2000).

Es gibt verschiedene Typen von Links.
absolut
HS-Anhalt <a href="http://www.hs-anhalt.de/index.html">HS-Anhalt</a>
lokales Projekt <a href="/project/content/index.html">lokales Projekt;/a>
relativ
hello world <a href="./hello_world.txt"></a>

Um etwaige Umzüge der Webseiten zu erleichtern, sind relative Links in der Regel am zweckmäßigsten.

Um Teile des Textes hervorzuheben wird für starke (engl. strong) Hervorhebungen <strong> genutzt, für schwächere <em> (von (von engl. emphasize). Abkürzungen können mit <abbr> (von engl. abbreviation) ausgezeichnet werden.

Absätze

Der Tag <p> (von engl. paragraph) kennzeichnet Absätze. Die folgenden typographischen Satzarten sind nicht auf <p>-Tags beschränkt, werden aber häufig dort eingesetzt.

Randnotiz

Top 10 Worte einer HTML-Datei

$> perl -np -e 's/<.*?>//g; s/^\s*//; s/\s+/\n/;' | sort | uniq -c | sort -nr | head

Beschreibung

Das Problem wurde mit Hilfe einer Verkettung von Filtern gelöst.

  1. Der erste Filter "perl" arbeitet zeilenweise und besteht aus drei Unterfiltern:
    1. Entfernen aller Tags (<.*?>)
    2. jeder "whitespace" (" ", \t, \n) am Anfang einer Zeile wird entfernt (^\s*)
    3. ein oder mehrere "whitespace" werden zu einem Zeilenumbruch (\s+/\n)
    Ausgabe: Ein Wort pro Zeile
  2. Sortieren (sort)
  3. Identische aufeinanderfolgende Zeilen werden zusammengefasst und mit deren vorangestellter Anzahl ausgegeben (uniq -c).
  4. Numerisches Sortieren in absteigender Reihenfolge (sort -nr)
  5. Ausgabe der ersten zehn Zeilen (head)

Diskussion

Alternativ zum "perl"-Filter hätten auch mehrere "sed" verwendet werden können.

Der Ansatz HTML-Tags auszufiltern ist vergleichsweise naiv, bspw. funktioniert er nicht bei auf mehrere Zeilen verteilten oder auskommentierten Tags. Ein in dieser Hinsicht robusterer, wenn auch langsamerer, Ansatz wäre der Einsatz eines Parsers.

Tabellen <table>

Tabellen <table> beinhalten Tabellen-Reihen (engl. table row) <tr> und diese Tabellen-Felder (engl. table data) <td>.

Tabellen-Felder können sich über mehrere Felder erstrecken, um dies zu erreichen werden die Attribute colspan für horizontal und rowspan für vertikale Zusammenfassung angegeben.

Code für eine Tabelle mit zwei Zeilen und zwei Spalten, die Tabelle nutzt in "Bootstrap" definierte CSS-Klassen. 
<table class="table table-bordered">
	<tr>
		<td>
			Feld 1/1
		</td>
		<td>
			Feld 1/2
		</td>
	</tr>
	<tr>
		<td colspan="2">
			Feld 2/1
		</td>
		<td>
			Feld 2/2
		</td>
	</tr>
</table>
Darstellung zum vorangegangenen Code:
Feld 1/1 Feld 1/2
Feld 2/1 Feld 2/2
