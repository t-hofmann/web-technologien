HTML

Bei HTML handelt es sich um eine Auszeichnungssprache. Teile eines Textes werden mit Hifle sogenannter Tags ausgezeichnet. Zwischen Starttag <TAGNAME> und Endtag </TAGNAME> steht der auszuzeichnende Text. Das Starttag kann zusätzlich Attribute <TAG ATTRIBUT="WERT"> enthalten, - den meisten Attributen muss ein Wert zugeordnet werden. Das class-Attribut verweist auf eine oder mehrere dem aktuellen Tag zugeordnete CSS-Klassen.

HTML-Kommentare <!-- Kommentar --> werden vom Browser nicht für die Darstellung herangezogen.

Die verschiedenartigen Elemente eines Hypertextes, wie bspw. Überschriften, Absätze, Listen, Tabelle, usw., können mit Hilfe dieser Tags ausgezeichnet werden. Im Folgenden wird ein einführender Überblick gegeben.

Ein html5-konformes Grundgerüst, - die erste Zeile <!DOCTYPE html> weist den Browser dabei auf html5 hin:

<!DOCTYPE html>
<html>
	<head>
		<!-- Metainformationen -->
	</head>
	<body>
		<!-- Darzustellender Inhalt -->
	</body>
</html>

Im head werden Metainformationen zum aktuellen Dokument untergebracht, während im body der darzustellende Inhalt steht.

Ligaturen

Eine Spezialität aus dem Schriftsatz sind sogenannte Ligaturen (mittellat. ligatura "Verbindung"). Dabei handelt es sich um spezielle Schnitte um benachbarte Buchstaben zu verbinden. Durch den Einsatz von Ligaturen soll ein harmonisches Schriftbild und eine gute Lesbarkeit erreicht werden. An dieser Stelle kann nicht vertiefend auf dieses Thema eingegangen werden, deshalb sei auf den Wikipedia-Artikel: "Ligatur (Typografie)" verwiesen.

Durch den Einsatz von &zwj; ("zero width joiner") bzw. &zwnj; ("zero width non joiner") kann eine Ligatur erlaubt bzw. verhindert werden. Das folgende Beispiel setzt voraus, dass die Schriftart "Palatino Linotype" installiert ist. Für eigene Experimente könnte die Suche innerhalb der Gruppe der sogenannten "gebrochenen" Schriften hilfreich sein.

Verweise

Wesentlich für HTML sind Verbindungen (engl. link). Ein Link-Ziel kann eine beliebige i. d. R. über http erreichbare Resource sein, - https im Falle der verschlüsselten Variante. Das Hypertext Transfer Protocol -- HTTP/1.1 (Roy Fielding, et al.; RFC 2616; IETF; 1999) ist im RFC 2616 definiert. Vertiefende Einblicke liefert in diesem Kontext die Dissertation des Hauptautors "Architectural Styles and the Design of Network-based Softeware Architectures" (Diss.; Roy Thomas Fielding; 2000).

Es gibt verschiedene Typen von Links.
absolut
HS-Anhalt <a href="http://www.hs-anhalt.de/index.html">HS-Anhalt</a>
lokales Projekt <a href="/project/content/index.html">lokales Projekt;/a>
relativ
hello world <a href="./hello_world.txt"></a>

Um etwaige Umzüge der Webseiten zu erleichtern, sind relative Links in der Regel am zweckmäßigsten.

Randnotiz

Top 10 Worte einer HTML-Datei

$> perl -np -e 's/<.*?>//g; s/^\s*//; s/\s+/\n/;' | sort | uniq -c | sort -nr | head

Beschreibung

Das Problem wurde mit Hilfe einer Verkettung von Filtern gelöst.

  1. Der erste Filter "perl" arbeitet zeilenweise und besteht aus drei Unterfiltern:
    1. Entfernen aller Tags (<.*?>)
    2. jeder "whitespace" (" ", \t, \n) am Anfang einer Zeile wird entfernt (^\s*)
    3. ein oder mehrere "whitespace" werden zu einem Zeilenumbruch (\s+/\n)
    Ausgabe: Ein Wort pro Zeile
  2. Sortieren (sort)
  3. Identische aufeinanderfolgende Zeilen werden zusammengefasst und mit deren vorangestellter Anzahl ausgegeben (uniq -c).
  4. Numerisches Sortieren in absteigender Reihenfolge (sort -nr)
  5. Ausgabe der ersten zehn Zeilen (head)

Diskussion

Alternativ zum "perl"-Filter hätten auch mehrere "sed" verwendet werden können.

Der Ansatz HTML-Tags auszufiltern ist vergleichsweise naiv, bspw. funktioniert er nicht bei auf mehrere Zeilen verteilten oder auskommentierten Tags. Ein in dieser Hinsicht robusterer, wenn auch langsamerer, Ansatz wäre der Einsatz eines html-Parsers.

Mit CSS ist es möglich, die beabsichtigte Darstellungsweise von HTML-Seiten zu beschreiben.

Box-Modell

Jedes Element einer HTML-Seite ist eine solche Box. Wobei es zwei Untertypen gibt, nämlich "inline"- und "block"-Elemente. Inline-Elemente stehen innerhalb des Fließtextes, während Block-Elemente typischerweise durch Zeilenumbrüche eingegrenzt sind.

Das Box-Modell wird durch folgende drei Kompontenten definiert.

margin
Außenabstand (grün)
border
Umrandung (schwarz)
padding
Innenabstand (gelb)

Zusätzlich kommt hier ein div-Element in "rot" als Inhalt der Box zum Einsatz.

Farben

Farben werden typischerweise mit Hilfe von RGB-Farbcodes definiert. Die einzelnen Farbkanäle können dabei Werte von 0 bis 255 annehmen.

rgb
rgb([0..255], [0..255], [0..255]); bspw. rgb(100, 90, 220)
rgba
Mit Hilfe von rgba kann zusätzlich zur Farbe ein Wert fü den zusätzlichen Alphakanal festgelegt werden, damit ist es möglich eine Farben durchscheinend zu machen. Die Werte des Alphakanals werden mit einer Dezimalzahl von 0 (vollkommen transparent) bis 1.0 (nicht transparent, opaque).
#
Die Raute # ermöglicht es den RGB-Farbcode hexadezimal anzugeben, dabei sind zwei Schreibweisen möglich: #fff ist bspw. äquivalent zu #ffffff;
named
Viele Farben sind über ihre Namen ansprechbar, etwa:
white lightgrey grey black
red darkred
orange darkorange
yellow lightyellow
green lightgreen darkgreen lime
blue lightblue darkblue
purple
Beispiele
Weiß: #fff, #ffffff, rgb(255,255,255), rgba(255,255,255,0), white
Schwarz: #000, #000000, rgb(0,0,0), rgba(0,0,0,0), black
